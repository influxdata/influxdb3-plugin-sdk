# Reference

The reference section documents the public file formats used by the SDK.

| Format | Purpose | Status |
|---|---|---|
| [`manifest.toml`](./manifest.md) | Describes one plugin version, including name, version, triggers, URLs, and dependencies. | Current |
| [`index.json`](./registry-index.md) | Describes one plugin registry, including artifact URLs, plugin versions, hashes, and yank state. | Current |

Reference pages are factual schema descriptions. They avoid workflow instructions and rationale so integrators can treat them as stable contracts.

For task-oriented setup guidance, start with [Getting Started](../01-getting-started/).
